# Ubiquitous Language

This glossary defines the domain terms used in the TD experiment repository. Use these terms consistently in code, scripts, reports, and review notes.

## Experiment Domain

| Term | Definition | Aliases to Avoid |
| --- | --- | --- |
| **TD Instance** | A complete finite-state TD(0) problem consisting of transition matrix, reward matrix, discount, stationary distribution, feature matrix, and derived objective quantities. | problem, environment, case |
| **Environment Family** | A named generator such as `toyexample` or `E10` that can produce one or more TD Instances by sweeping parameters. | env, experiment, benchmark |
| **Case** | One concrete TD Instance produced from an Environment Family and one parameter assignment. | omega, scale, instance row |
| **Omega Level** | The value of the TD matrix conditioning quantity `omega` recorded in `manifest.tsv` for one Case. | omega case, eigen case |
| **Schedule** | A learning-rate formula selected by name, such as `theory`, `inv_t`, or `constant_omega`. | stepsize, method |
| **Projection** | The TD parameter clipping policy selected by name: `none`, `oracle`, or `upper`. | projected TD, radius setting |
| **Method** | One Schedule and one Projection considered together. | algorithm, TD algorithm |
| **C Sweep** | The set of positive scalar `c` values used in learning-rate denominators. | base values, scaling grid |
| **Run** | One Monte Carlo trajectory for a fixed Case, Method, and `c`. | seed, replica |
| **Sweep** | The Cartesian product over Cases, Schedules, Projections, `c` values, and Runs launched by `./cpp/tdx sweep`. | experiment, full run |
| **Exactness Baseline** | A trusted output set used as the byte-for-byte reference for `manifest.tsv`, `runs_case_*.csv`, and `agg_case_*.csv`. | old result, ground truth |

## Output and Reporting

| Term | Definition | Aliases to Avoid |
| --- | --- | --- |
| **Run Directory** | One timestamped output directory for a single Environment Family sweep. | log folder, env folder |
| **Manifest** | The `manifest.tsv` file that indexes all Cases, Methods, `c` values, metrics, and output CSV filenames in a Run Directory. | metadata file, index |
| **Manifest Row** | One row in `manifest.tsv` for exactly one Case, Method, and `c` combination. | experiment row, config row |
| **Run CSV** | A `runs_case_*.csv` file containing per-Run final metrics and divergence information for one Manifest Row. | seed CSV, MC CSV |
| **Aggregate CSV** | An `agg_case_*.csv` file containing checkpoint-averaged learning curves for one Manifest Row. | curve CSV, average CSV |
| **Plot Suite** | The complete set of PNG figures generated by `scripts/plot_suite_v2.py` for one Run Directory. | figures, plots |
| **Embedded Report** | A self-contained HTML report generated by `scripts/generate_embedded_report_v2.py` with images embedded as data URIs. | HTML report, gallery |

## Metrics and Quantities

| Term | Definition | Aliases to Avoid |
| --- | --- | --- |
| **Suboptimality D** | The checkpoint objective stored as `E_D[||Vbar_t - V*||^2]` in aggregate CSVs. | D error, final error |
| **Suboptimality D+A** | The checkpoint objective stored as `E_A[||Vbar_t - V*||^2]` in aggregate CSVs. | A error, D plus A |
| **Divergence Rate** | The fraction of Runs whose squared norm exceeded the divergence threshold or became non-finite. | failure rate |
| **Ratio** | The per-Run value `max_theta_norm / max(theta_star_norm_sq, 1e-16)` recorded in Run CSVs. | norm ratio |
| **Omega** | The minimum eigenvalue-like TD conditioning quantity recorded as `omega` in Manifest and Aggregate CSVs. | lambda, eig |
| **Symmetric Lambda** | The minimum eigenvalue of `(A+A^T)/2`, recorded as `lambda_sym`. | omega, A eigenvalue |
| **Tau Proxy** | The transition-chain mixing proxy recorded as `tau_proxy`. | mixing time |
| **Phi Infinity Squared** | The maximum squared feature-row norm recorded as `phi_max_sq`. | feature norm |
| **Theta Star Norm** | The Euclidean norm of the exact TD fixed point `theta*`, recorded as `theta_star_norm`. | oracle radius |

## Relationships

- One **Environment Family** produces one or more **Cases**.
- One **Case** plus one **Method** plus one `c` value produces one **Manifest Row**.
- One **Manifest Row** points to exactly one **Run CSV** and one **Aggregate CSV**.
- One **Sweep** contains many **Runs**, but a **Run CSV** summarizes all Monte Carlo Runs for one **Manifest Row**.
- **Omega** and **Symmetric Lambda** are distinct quantities and must not be used interchangeably.
- **Projection** is part of a **Method**; `oracle` uses **Theta Star Norm**, while `upper` uses the theoretical radius bound.

## Example Dialogue

> **Developer:** "For `E10`, should I call each omega value an Environment?"
>
> **Domain Expert:** "No. `E10` is the **Environment Family**. Each swept parameter assignment is a **Case**, and each Case has an **Omega Level**."
>
> **Developer:** "So one **Manifest Row** is Case plus Schedule plus Projection plus `c`?"
>
> **Domain Expert:** "Yes. That row points to one **Run CSV** and one **Aggregate CSV**."
>
> **Developer:** "Can I treat **Omega** as the minimum eigenvalue of `(A+A^T)/2`?"
>
> **Domain Expert:** "No. Use **Symmetric Lambda** for `(A+A^T)/2`; **Omega** is the TD conditioning quantity recorded separately."

## Flagged Ambiguities

- "algorithm" has been used for both **Schedule** alone and **Method** as Schedule plus Projection. Use **Method** when Projection matters.
- "instance" has been used for both **Environment Family** and **Case**. Use **TD Instance** only for a concrete Case.
- "omega" has been used to refer to both **Omega** and **Symmetric Lambda**. Keep them separate in table headings, plot titles, and reports.
- "base value" and "scaling" both refer to the scalar `c`; prefer **C Sweep** for the grid and `c` for a single value.
